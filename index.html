HTML

  • HTML stands for HyperText Markup Language and is the standard markup language for creating web pages.
  • Tags in HTML define the structure of the content, such as headings, paragraphs, and lists.
  • Attributes provide additional information about HTML elements and are always included in the opening tag.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS, which stands for Cascading Style Sheets, is used for styling web pages written in HTML or XML.
  • Selectors in CSS target specific HTML elements to apply styles, such as colors, fonts, and spacing.
  • CSS can be applied internally within an HTML document, externally through a separate CSS file, or inline within specific HTML elements.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a distributed version control system that tracks changes in source code during software development.
  • Repositories in Git store the entire history of a project, allowing collaboration among multiple developers.
  • Commands like git add, git commit, and git push are used to stage, record, and share changes, respectively.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.

JavaScript

  • JavaScript is a high-level, interpreted programming language that adds interactivity and dynamic behavior to web pages.
  • Functions in JavaScript allow the encapsulation of reusable pieces of code for better organization and maintainability.
  • JavaScript supports object-oriented, imperative, and functional programming paradigms.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
